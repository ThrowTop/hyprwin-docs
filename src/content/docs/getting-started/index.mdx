HyprWin is a portable Windows 11 x64 application. Windows 10 may work but is
untested and unsupported. HyprWin does not require an installer or a fixed
installation directory.

## Install

<LinkButton
  href="https://github.com/ThrowTop/hyprwin/releases"
  icon="download"
>
  Download from GitHub Releases
</LinkButton>

Download the current `windows-x64` ZIP and extract it into a writable folder.
Release notes and older versions remain available on the same page. The core
runtime layout is:

<FileTree>

- HyprWin
  - **hyprwin.exe** the application
  - **lua51.dll** the required LuaJIT runtime
  - **d3dcompiler_47.dll** runtime custom-shader compiler
  - shaders/
    - **hyprwin_shader_api.hlsl** custom-shader contract
  - **hyprwin.lua** the active configuration

</FileTree>

`hyprwin.lua` is created from the embedded default configuration on first
launch.

Release archives may also contain `.luarc.json` and `hw_annotations.lua`.
These provide Lua Language Server completion and type information and are not
required at runtime.

## First launch

<Steps>

1. Run `hyprwin.exe`.

   Release builds request administrator privileges and Windows may show a User
   Account Control prompt.

2. Find HyprWin in the system tray.

   The tray menu provides **Reload Config**, **Open Config Folder**, and
   **Exit**.

3. Open the configuration folder from the tray menu.

   Only one HyprWin instance can run at a time.

</Steps>

## How the controls work

The configured SUPER key is `LWIN` by default.

- Hold SUPER and press a configured key to run a Lua binding.
- Hold SUPER and drag with the left mouse button to move a window.
- Hold SUPER and drag with the right mouse button to resize a window.

## Add a binding

Open `hyprwin.lua` and add:

```lua title="hyprwin.lua"
hw.bind("RETURN", function()
  hw.launch("wt.exe")
end)
```

Reload the configuration from the tray menu. Holding SUPER and pressing Enter
now launches Windows Terminal.

Binding strings omit `SUPER`, so `"RETURN"` means SUPER+Enter.

## Generated folders

HyprWin may create these folders beside the executable:

<FileTree>

- HyprWin
  - logs/ timestamped run logs
  - shader-cache/ cached custom shader bytecode

</FileTree>

HyprWin retains up to five current log files.
