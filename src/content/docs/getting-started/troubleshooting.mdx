Start with the [HyprWin log](/hyprwin-docs/logging/). It records startup,
configuration, binding, shader, and native subsystem failures.

## HyprWin does not start

- Confirm that `lua51.dll` is beside `hyprwin.exe`.
- Accept the administrator elevation request.
- Check whether another HyprWin instance is already running.

## Configuration error message

Open the newest file under `logs/`, fix the reported error, then select
**Reload Config**. See [Configuration errors and safe
mode](/hyprwin-docs/getting-started/configuration/#configuration-errors-and-safe-mode)
for the recovery behavior.

## A binding does not run

<Steps>

1. Hold the configured SUPER key while pressing the binding key.
2. Do not include `SUPER` in the `hw.bind()` key string.
3. Confirm the configuration reloaded successfully.
4. Use `hw.binds()` to inspect the registered side-specific combinations.
5. Check the log for callback errors or the 50 ms Lua watchdog.

</Steps>

Enable binding diagnostics:

```lua title="hyprwin.lua"
hw.settings.debug = {
  "trace_binds",
  "bench_binds",
}
```

`PAUSE` and `PRINTSCREEN` are understood by the parser but are not dispatched
to bindings by the current low-level hook.

## Mouse drag or resize ignores a window

HyprWin rejects targets that are minimized, invisible, cloaked, child windows,
tool windows, non-activating windows, unresponsive, or borderless fullscreen.
Right-button resize also requires a resizable window.

Enable selection logging:

```lua title="hyprwin.lua"
hw.settings.debug = { "trace_grabs" }
```

Review the logged rejection reason and `hw.settings.grab_filters`.

## Thumbnail preview remains border-only

Thumbnail mode starts with the normal border overlay while Windows Graphics
Capture requests permission and waits for its first frame. It remains
border-only if capture is unsupported, permission is denied, capture fails, or
no frame arrives within 100 ms after capture starts. In that fallback, HyprWin
does not park the real window.

Check the log for `window_snapshot` diagnostics. Permission is requested once
per HyprWin process; restart HyprWin after changing the Windows capture
permission.

## A custom shader is not active

- `hw.settings.shader` resolves to the intended file.
- `d3dcompiler_47.dll` is available beside HyprWin or from System32.
- The shader includes `hyprwin_shader_api.hlsl`.
- It implements `hyprwin_pixel`.
- It returns premultiplied RGBA.
- The log contains no compile warning or error.

```lua title="hyprwin.lua"
local status = hw.debug.shader_compiler_status()
hw.log("info", status.diagnostics)
```

If only an included HLSL file changed, touch the main shader file or clear its
cache files under `shader-cache/`.
